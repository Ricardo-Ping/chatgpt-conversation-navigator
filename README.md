# ChatGPT Branch Map

一个面向 `chatgpt.com` / `chat.openai.com` 的 Chrome 扩展原型，用来把长对话中的关键节点标出来，并以“脑图 + 树形”双视图快速回溯。

## 当前版本能做什么

- 鼠标移动到某条消息上时，显示“设为节点”按钮。
- 将某条消息保存为分支节点。
- 默认把新节点挂到当前选中的节点下面，形成分支关系。
- 支持“脑图视图”：节点卡片 + 曲线连线 + 自由拖拽。
- 支持“树形视图”：列表方式浏览所有节点。
- 拖拽后的节点位置会自动持久化保存。
- 支持“建树”：从当前页面消息自动生成基础树结构。
- 支持节点折叠/展开，适合长对话收拢分支。
- 支持“同页开分支”：从选中节点发起分支，不弹新窗口。
- 支持“返回父会话”：分支会话可一键回到来源会话节点。
- 支持节点搜索（标题/摘要/内容关键字）。
- 支持“消息导航栏”：顶部/底部、上一节点/下一节点、时间轴点位跳转。
- 支持 Mini Map 快速定位大画布节点。
- 支持自动刷新开关（监听新消息自动更新）。
- 支持拖动右下角手柄调整面板大小。
- 支持拖拽改父子关系（将节点拖到另一个节点上重组分支）。
- 支持右侧固定极简模式。
- 点击节点后自动滚动到原始消息位置并高亮。
- 一键复制“继续这个分支”的提示词，方便你把问题重新拉回到某个节点。
- 按对话 URL 维度持久化保存，不同聊天互不影响。

## 这个版本还不能做什么

- 不能真正修改 ChatGPT 服务器端已经积累的上下文。
- 不能像本地知识图谱一样真正切断模型记忆，只能帮助你“定位并重启一个分支”。
- 同页开分支依赖页面结构，若 ChatGPT 前端大改，可能需要适配选择器。

## 安装方式

1. 打开 Chrome 扩展页：`chrome://extensions/`
2. 开启右上角“开发者模式”
3. 点击“加载已解压的扩展程序”
4. 选择你本地克隆/下载后的项目根目录（包含 `manifest.json` 的目录）

5. 打开 ChatGPT 对话页面并刷新

## 使用方式

1. 在一条你想作为“主线/分支起点”的消息上点击“设为节点”
2. 当你读到某个问题需要岔出去时，把岔出去前的消息设为节点
3. 后续继续设节点时，会自动接到当前选中的节点下
4. 头部按钮可切换“脑图/树形”，脑图中可直接拖动节点排版
5. 如需快速建结构，点击“建树”自动生成基础树
6. 点击“开分支”可在当前页发起分支聊天；点击“返回父”可回到来源会话
7. 可用搜索框按关键词过滤节点
8. 使用导航栏在长对话里快速跳转（顶部/底部/节点前后/时间轴）
9. 脑图中把节点拖到另一个节点上可重组父子关系
10. 点击“极简”进入右侧固定极简模式，减少遮挡
11. 想回到某个分支时，点击节点里的“跳到消息”
12. 想重新发问时，点击“复制继续”，再把提示贴进输入框
